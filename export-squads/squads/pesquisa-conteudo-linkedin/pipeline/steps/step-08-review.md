# Step 08: Revisão do Post

## Instrução

Aplicar o checklist completo de revisão ao post rascunho do step-07.

### Processo:

1. Ler o post completo em `output/post-draft.md`
2. Aplicar os 4 blocos do checklist (Hook 40%, Estrutura 25%, Tom 20%, CTA 15%)
3. Emitir veredicto claro

### Formato de entrega:

```
## Revisão — [Data]

### 1. Hook (40%) — [✅ OK | ❌ FALHA]
- [ ] ≤ 210 caracteres: [contagem real]
- [ ] Para o scroll: [Sim/Não — justificativa]
- [ ] Estrutura validada: [qual estrutura usou]
- [ ] Sem clichê: [Sim/Não]

### 2. Estrutura (25%) — [✅ OK | ❌ FALHA]
- [ ] Parágrafos max 2 linhas: [Sim/Não]
- [ ] Espaço branco entre blocos: [Sim/Não]
- [ ] Uma ideia central: [Sim/Não]
- [ ] Fluxo lógico: [Sim/Não]
- [ ] ≤ 1.300 chars: [contagem real]

### 3. Tom de Voz (20%) — [✅ OK | ❌ FALHA]
- [ ] Linguagem coloquial BR: [Sim/Não]
- [ ] Anti-guru: [Sim/Não]
- [ ] Sem vocabulário proibido: [Sim/Não — listar se houver]
- [ ] Sem emojis emocionais: [Sim/Não]

### 4. CTA (15%) — [✅ OK | ❌ FALHA]
- [ ] Ação específica ou pergunta: [Sim/Não]
- [ ] Não vende diretamente: [Sim/Não]
- [ ] Gera conversa: [Sim/Não]

---

## Veredicto: ✅ APROVADO | ❌ REJEITADO

### Feedback (se rejeitado):
| # | Problema | Por quê | Sugestão |
|---|----------|---------|----------|
| 1 | [problema] | [motivo] | [como corrigir] |

### Recomendação:
[1-2 frases com o caminho para aprovação]
```

### Fluxo pós-revisão:

- **✅ APROVADO** → Salvar em `output/post-final.md` → Avança para step-09
- **❌ REJEITADO** → Feedback em `output/review-feedback.md` → Volta para step-07 (redator corrige)

## Output
Revisão salva em `output/review-feedback.md` ou post aprovado em `output/post-final.md`
