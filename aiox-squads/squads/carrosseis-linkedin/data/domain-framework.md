# Domain Framework — Carrosséis LinkedIn B2B

## Metodologia Operacional

### 1. Receber Tema
Entender o tema fornecido pelo usuário. Classificar pela fonte de tese (Skills em Produção, Benchmark Real, Process Diagnostic, Falha Documentada).

### 2. Gerar Ângulos
A partir do MESMO tema, criar 5 ângulos distintos usando drivers psicológicos diferentes:
- 🔴 Medo/Perda: "Isso vai custar caro se você ignorar"
- 🟢 Oportunidade: "Sua janela antes que todo mundo descubra"
- 📚 Educacional: "Testei X — veja o que aconteceu"
- ↔️ Contrário: "O que ninguém te fala sobre X"
- ⭐ Inspiracional: "Imagine o cenário onde isso funciona"

### 3. Selecionar Ângulo
O usuário escolhe O MELHOR ângulo para o momento. O ângulo define tom, estrutura e CTA.

### 4. Criar Copy Slide-a-Slide
Com o ângulo selecionado:
1. Escrever hook (slide 1) — max ~210 chars antes do fold
2. Criar 3-8 slides de conteúdo (1 ideia por slide, max 30 palavras)
3. Fechar com CTA (pergunta genuína ou ação específica)
4. Adicionar 3-5 hashtags na última linha do post

### 5. Selecionar Tom de Voz
Ler `tone-of-voice.md`, recomendar o tom mais adequado ao ângulo e à lente "Built, not prompted", apresentar as 6 opções ao usuário.

### 6. Escolher Estilo Visual
Com base no tema e ângulo:
- **Twitter-Style:** Melhor para temas que envolvem prints, notícias, dados de terceiros, autoridade
- **Pessoa-Style:** Melhor para temas pessoais, jornada, bastidores, conexão humana

### 7. Criar Design Visual
Seguir o estilo selecionado (ver `visual-styles.md`):
- Definir design system
- Criar HTML/CSS self-contained para cada slide
- Renderizar e verificar

### 8. Revisar
Checklist rápido focado em: tom, visual, CTA, aderência à lente, consistência.
